The list endpoints expose paginated catalog browsing.

Endpoint references:

- [`listSongs`](/openmusic/endpoints/listSongs)
- [`listAlbums`](/openmusic/endpoints/listAlbums)
- [`listArtists`](/openmusic/endpoints/listArtists)
- [`listPlaylists`](/openmusic/endpoints/listPlaylists)

Available list endpoints:

- [`List Songs`](./list-songs)
- [`List Albums`](./list-albums)
- [`List Artists`](./list-artists)
- [`List Playlists`](./list-playlists)

Shared behavior:

- `limit` and `offset` control pagination
- `sort` is supported where the server advertises the `sort` capability
- `filter[genres]` narrows the result set
- `include` and scoped `view[...]` parameters shape nested relationships
