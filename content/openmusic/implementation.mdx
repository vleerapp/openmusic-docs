# Get Started

To get started on implementing your own server take a look at what capabilities are and why you should implement as much of them as possible.

# Capabilities

Capabilities are a way for a server to communicate to the client what it can and can not do.
Some may be required to provide basic functionallity while others stay optional for servers to implement.
All the others are optional but should be implemented by the client to ensure compatibility.

<Callout title="Info" type="info">
  Each capability has its own respecting API endpoints.
</Callout>

| Feature                                      | Type     | Description                             |
| -------------------------------------------- | -------- | --------------------------------------- |
| [`info`](/openmusic/endpoints/getInfo)       | Required | Info about the server                   |
| [`search`](/openmusic/endpoints/searchSongs) | Required | Search songs by string                  |
| [`song`](/openmusic/endpoints/getSong)       | Required | Fetch metadata for song by id           |
| `download`                                   | Required | Download song                           |
| `stream`                                     | Optional | Stream song                             |
| `artists`                                    | Optional | Search and get artists                  |
| `albums`                                     | Optional | Search and get albums                   |
| `playlists`                                  | Optional | Search and get playlists                |
| `stations`                                   | Optional | Search and get stations                 |
| `sort`                                       | Optional | Sort by dateReleased, dateAdded or name |
| `lyrics`                                     | Optional | Lyrics                                  |
| `list`                                       | Optional | Paginate all available types            |

## General

### Query Param Definitions

| Name            | Description                                   | Type                                        | Example                  |
| --------------- | --------------------------------------------- | ------------------------------------------- | ------------------------ |
| `view`          | Strip attributes and relationships            | `"minimal"`                                 | `view=minimal`           |
| `include`       | Include relationships                         | `string[]`                                  | `include=artists,albums` |
| `limit`         | Limit response items                          | `number`                                    | `limit=10`               |
| `offset`        | The offset for paginated responses            | `number`                                    | `offset=10`              |
| `sort`          | Sort by. Add a `-` before to sort descending. | `"-?dateReleased"\|"-?dateAdded"\|"-?name"` | `sort=-dateReleased`     |
| `filter[scope]` | Filter for specific scopes                    | `string`                                    | `filter[genres]=pop`     |

Query Params can be scoped.
If for example a `song` is fetched with `include=albums` the `albums` can be requested with the minimal view by adding `view[albums]=minimal`


### Get a Song

URL

`GET /songs/{id}`

Path Params:

| Name | Description | Type     | Example |
| ---- | ----------- | -------- | ------- |
| `id` | Song id     | `string` | `1`     |

Query Params:

| Name      | Description                             | Type              | Example                          |
| --------- | --------------------------------------- | ----------------- | -------------------------------- |
| `ids`     | Get by ids                              | `string[]`        | `ids=1,2`                        |
| `isrc`    | Get by isrc ids                         | `string[]`        | `isrc=JPU902504960,JPU902504961` |
| `view`    | [`Read more`](#query-param-definitions) |                   |                                  |
| `include` | [`Read more`](#query-param-definitions) | `albums, artists` |                                  |

Example Request:



```json lineNumbers
{
  "id": "abc123def456",
  "title": "Example Song Title",
  "artistName": "Example Artist",
  "releaseYear": 2024,
  "coverURL": "/song/abc123def456/art"
}
```

## Stream

The input should be a song id which has been returned by either [listSongs](#songs) or in from the [search](#search) results.

The return is a file directly streamed in the native format it was saved in using 206 Partial Content responses with the byte range.

```yml lineNumbers
Accept-Ranges: bytes
Content-Length: 6272474
Content-Type: audio/mpeg
```

## Cover

The input should be a song id which has been returned by either [listSongs](#songs) or in from the [search](#search) results.

The return should be a square jpeg downscaled to 512x512 to ensure fast loading speeds with high enough quality.

```yml lineNumbers
Content-Type: image/jpeg
Content-Length: 73618
```

## Songs

The return should be a list of all songs the server serves this may optionally have multiple pages.

```json lineNumbers
[
  {
    "id": "abc123def456",
    "title": "Example Song Title",
    "artistName": "Example Artist",
    "releaseYear": 2024,
    "coverURL": "/song/abc123def456/art"
  },
  {
    "id": "xyz789ghi012",
    "title": "Another Song Example",
    "artistName": "Another Artist",
    "releaseYear": 2023,
    "coverURL": "/song/xyz789ghi012/art"
  }
]
```

## Search

The input should be a string.

The return is a list of all matching results from the input query string. The results are matched based on the title, artist and album.

```json lineNumbers
[
  {
    "id": "abc123def456",
    "title": "Example Song Title",
    "artistName": "Example Artist",
    "releaseYear": 2024,
    "coverURL": "/song/abc123def456/art"
  }
]
```
